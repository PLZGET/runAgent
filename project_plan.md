# 🚀 퇴사 에이전트 "탈출 마스터"

> "나 퇴사하고 싶어" 한마디면, 퇴사의 A to Z를 브라우저 액션으로 실제 수행하는 AI 에이전트

---

## 1. 프로젝트 개요

### 1-1. 문제 정의

퇴사는 인생에서 가장 복잡한 의사결정 중 하나다.

- 퇴직금, 실업급여, 4대보험 등 **법률/재무 지식**이 필요
- 이직 공고 탐색, 자소서 작성 등 **시간 소모가 큼**
- 정보가 여러 사이트에 **파편화**되어 있음 (고용보험, 채용사이트, 잡플래닛 등)
- 심리적 부담으로 **체계적 준비 없이 감정적으로 퇴사**하는 경우 다수

### 1-2. 솔루션

사용자의 상황을 파악하고, **실제 브라우저를 조작하여** 정보 수집 → 분석 → 서류 생성 → 이직 준비까지 End-to-End로 수행하는 에이전트.

### 1-3. 핵심 차별점

| 기존 서비스 | 탈출 마스터 |
|---|---|
| 퇴직금 계산기 (단일 기능) | 퇴사 전 과정을 하나의 흐름으로 |
| 채용사이트 (검색만) | 조건 분석 후 최적 공고 추천 |
| 블로그 정보 (읽기만) | 실제 사이트 접속하여 자동 수행 |
| 각각 따로 해야 함 | 한 에이전트가 전부 처리 |

## 2. 에이전트 파이프라인

### Phase 1: 퇴사 진단 🔍

```
사용자 입력: "나 퇴사하고 싶어"
    ↓
[상황 파악 Agent]
    - 대화형 인터뷰 (현재 연봉, 근속기간, 퇴사 사유, 희망 직무 등)
    - "진짜 퇴사할 때 맞는지" 체크리스트 제공
    - 퇴사 타이밍 분석 (성과급, 연차, 퇴직금 구간 고려)
    ↓
[결과] 퇴사 리포트 v1 생성
```

**수집 정보:**
- 입사일, 현재 연봉, 직무, 산업군
- 퇴사 희망 시기
- 희망 연봉, 희망 직무/산업
- 현재 보유 스킬

### 3-2. 상세 기술 스택

| 구분 | 기술 | 선택 이유 |
|---|---|---|
| **Frontend** | Next.js 14 + TailwindCSS | 빠른 개발, SSR 지원 |
| **실시간 통신** | WebSocket (Socket.io) | 브라우저 액션 실시간 스트리밍 |
| **Backend** | FastAPI (Python) | 비동기 처리, LLM 연동 용이 |
| **LLM** | Claude API (Sonnet) | 추론 능력, 한국어 성능 |
| **브라우저 자동화** | Playwright (Python) | 안정성, 멀티브라우저, 스크린샷 |
| **에이전트 프레임워크** | LangGraph | 멀티스텝 워크플로우, 상태 관리 |
| **DB** | SQLite (해커톤용) | 세팅 최소화 |
| **배포** | Docker Compose | 원클릭 실행 환경 |

### 3-3. 브라우저 액션 구현

```python
# 예시: Playwright로 원티드 검색 자동화
async def search_wanted(query: str, filters: dict):
    async with async_playwright() as p:
        browser = await p.chromium.launch(headless=False)  # headless=False로 데모 시 화면 보여줌
        page = await browser.new_page()

        # 1. 원티드 접속
        await page.goto("https://www.wanted.co.kr")

        # 2. 검색어 입력
        await page.fill('input[type="search"]', query)
        await page.press('input[type="search"]', 'Enter')

        # 3. 필터 적용 (연봉, 경력 등)
        await apply_filters(page, filters)

        # 4. 결과 수집
        jobs = await extract_job_listings(page)

        # 5. 각 액션마다 스크린샷 → 프론트엔드로 스트리밍
        screenshot = await page.screenshot()
        await stream_to_frontend(screenshot)

        return jobs
```

### 3-4. 에이전트 오케스트레이션 (LangGraph)

```python
from langgraph.graph import StateGraph

# 상태 정의
class ExitPlanState(TypedDict):
    user_info: dict           # 사용자 정보
    diagnosis: dict           # 퇴사 진단 결과
    job_listings: list        # 수집된 공고
    legal_info: dict          # 법률/재무 정보
    documents: list           # 생성된 서류
    coaching: dict            # 코칭 내용
    current_phase: str        # 현재 단계

# 그래프 구성
workflow = StateGraph(ExitPlanState)

workflow.add_node("diagnosis", diagnosis_agent)
workflow.add_node("market_analysis", market_analysis_agent)
workflow.add_node("legal_check", legal_agent)
workflow.add_node("document_gen", document_agent)
workflow.add_node("job_prep", job_prep_agent)
workflow.add_node("coaching", coaching_agent)

# 흐름 정의
workflow.add_edge("diagnosis", "market_analysis")
workflow.add_edge("market_analysis", "legal_check")
workflow.add_edge("legal_check", "document_gen")
workflow.add_edge("document_gen", "job_prep")
workflow.add_edge("job_prep", "coaching")

app = workflow.compile()
```
