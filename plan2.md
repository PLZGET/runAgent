# 탈출 마스터 - 해커톤 실행 계획 v2

> 퇴사 전 손익과 이직 가능성을 실제 브라우저 액션으로 정리해주는 AI 에이전트

---

## 1. 한 줄 요약

사용자가 "나 퇴사하고 싶어"라고 말하면, 에이전트가 현재 상황을 진단하고 채용 사이트를 직접 탐색해 이직 가능성을 분석한 뒤, 바로 쓸 수 있는 퇴사 준비 문서를 생성한다.

핵심은 단순 상담이 아니라 `추론 + 브라우저 행동 + 결과물 생성`까지 한 흐름으로 보여주는 것이다.

---

## 2. 왜 이 문제인가

퇴사는 단순한 감정 문제가 아니다. 아래 3가지가 동시에 얽혀 있다.

- 재무: 퇴직금, 성과급, 공백 기간, 희망 연봉
- 정보 탐색: 채용 공고, 기업 평가, 시장 연봉
- 실행 준비: 사직서, 체크리스트, 이직 준비 자료

문제는 이 정보들이 각각 다른 사이트와 문서에 흩어져 있다는 점이다. 사용자는 퇴사를 고민하는 순간에도 실제로는 검색, 비교, 정리, 문서 작성까지 모두 수동으로 해야 한다.

이 프로젝트는 이 파편화된 과정을 하나의 에이전트 흐름으로 묶는다.

---

## 3. 제품 정의

### 제품명
탈출 마스터

### 핵심 가치

- 지금 퇴사해도 되는지 판단 근거를 준다
- 내 조건에서 이직 가능성이 어느 정도인지 빠르게 보여준다
- 조사 결과를 바로 행동 가능한 문서로 바꿔준다

### 차별점

| 기존 방식 | 탈출 마스터 |
|---|---|
| 블로그/커뮤니티를 직접 뒤진다 | 에이전트가 직접 사이트를 탐색한다 |
| 계산기, 채용 검색, 문서 작성을 따로 한다 | 하나의 흐름으로 연결한다 |
| 검색 결과를 사람이 다시 해석해야 한다 | 개인 조건 기준으로 우선순위를 정리한다 |
| 정보만 제공한다 | 결과물까지 생성한다 |

---

## 4. 해커톤용 범위 전략

이 프로젝트의 핵심은 기능 수가 아니라 `끝까지 닫히는 데모`다. 따라서 범위를 아래처럼 3단계로 분리한다.

### 4-1. MVP

심사장에서 반드시 실제 동작으로 보여줄 범위

- 사용자 입력
  - 입사일
  - 현재 연봉
  - 경력 연차
  - 현재 직무
  - 희망 직무
  - 희망 연봉
- 퇴사 진단
  - 현재 퇴사 시점의 손익 코멘트 생성
  - 성과급/퇴직금/공백 위험 등 rule-based 분석
- 브라우저 액션
  - 채용 사이트 1곳 live 탐색
  - 검색어 입력, 결과 페이지 이동, 공고 리스트 수집
- 공고 분석
  - 중복 제거
  - 기술 스택 키워드 매칭
  - 연봉/직무 기준 점수화
  - 추천 공고 Top 5 생성
- 결과물 생성
  - 퇴사 리포트 1종
  - 사직서 초안 또는 퇴사 체크리스트 1종

### 4-2. Stretch

시간이 남으면 붙이는 기능

- 채용 사이트 2~3곳 추가 수집
- 기업 리뷰/평판 데이터 결합
- 공고별 맞춤 자소서 초안 생성
- 면접 예상 질문 생성
- 연봉 협상 시뮬레이션

### 4-3. Narrative Only

발표에서는 말하되, 데모 성공이 여기에 걸리지 않게 두는 기능

- 실업급여 자격 정밀 판정
- 4대보험 후속 처리 가이드 자동화
- 감정 케어/상사 설득 스크립트
- 퇴사 후 첫 주 일정 코칭

## 5. 핵심 사용자 흐름

```text
사용자: "나 퇴사하고 싶어"
  ↓
Intake Agent
  - 현재 상황 인터뷰
  - 구조화된 프로필 생성
  ↓
Timing Agent
  - 퇴사 타이밍/손익 코멘트 생성
  ↓
Browser Agent
  - 채용 사이트 live 탐색
  - 조건 기반 공고 수집
  ↓
Ranking Agent
  - 공고 정규화
  - 매칭 점수 계산
  - Top 5 추천
  ↓
Document Agent
  - 퇴사 리포트 생성
  - 사직서 초안 또는 체크리스트 생성
```

중요한 점은 모든 단계를 fully autonomous로 만들 필요는 없다는 것이다. 해커톤 데모에서는 사용자가 최소한의 입력만 하고, 에이전트가 나머지 흐름을 이어서 처리하는 모습을 보여주면 충분하다.

---

## 6. 데모 전략

### 목표

5분 안에 아래 4가지를 명확히 보여준다.

- 자연어 요청 하나로 작업이 시작된다
- 에이전트가 실제 브라우저를 조작한다
- 수집 결과를 개인화된 추천으로 바꾼다
- 최종 결과물이 바로 나온다

### 권장 데모 시나리오

#### 0:00 - 0:40

사용자가 "나 퇴사하고 싶어" 입력  
에이전트가 간단한 추가 질문을 통해 프로필 완성

#### 0:40 - 1:40

현재 상황 분석 결과 표시

- "지금 퇴사하면 성과급 반영 전이라 손해 가능성 있음"
- "시장에서는 현재 연봉 대비 상향 여지 있음"

#### 1:40 - 3:00

브라우저 액션 시연

- 원티드 같은 채용 사이트 1곳 접속
- 검색어 입력
- 결과 페이지 이동
- 공고 목록 수집
- 실시간 로그/스크린샷 표시

#### 3:00 - 4:20

분석 결과 표시

- 수집 공고 수
- 추천 Top 5
- 연봉 상향 가능성
- 스킬 매칭 점수

#### 4:20 - 5:00

문서 출력

- 퇴사 리포트
- 사직서 초안 또는 퇴사 체크리스트

### 데모 안정성 원칙

- live는 `브라우저 액션 증명`에 집중한다
- 결과 분석은 `live + 캐시/Mock` 하이브리드로 받친다
- 심사장에서 반드시 모든 사이트를 실시간으로 돌릴 필요는 없다

즉, "브라우저를 실제로 조작하는 에이전트"를 보여주되, 결과 계산과 추천은 안정적으로 완주되게 설계한다.

## 7. 기술 설계

### 7-1. 아키텍처

```text
Next.js 대시보드
  - 채팅 입력
  - 진행 상태
  - 브라우저 액션 로그/스크린샷
  - 결과 리포트 표시

FastAPI 백엔드
  - 세션 관리
  - 에이전트 오케스트레이션
  - 문서 생성 API

Playwright
  - 브라우저 자동화
  - 검색/스크롤/리스트 추출

LLM
  - 인터뷰 질문
  - 진단 코멘트
  - 리포트/문서 생성

SQLite
  - 세션/결과 임시 저장
```

### 7-2. 권장 스택

| 영역 | 기술 | 이유 |
|---|---|---|
| Frontend | Next.js + TailwindCSS | 빠른 프로토타이핑, 대시보드 구현 용이 |
| Backend | FastAPI | 비동기 처리, Python 생태계 활용 |
| Browser | Playwright | 안정적인 브라우저 자동화 |
| LLM Orchestration | LangGraph 또는 단순 상태 머신 | 멀티스텝 흐름 관리 |
| DB | SQLite | 세팅 비용 최소화 |
| 배포 | Docker Compose | 실행 환경 통일 |

### 7-3. 구현 원칙

- 브라우저 액션은 1개 사이트부터 끝낸다
- 추론과 계산은 가능하면 구조화된 입력 기반으로 한다
- 법률/재무 정보는 `판정`보다 `참고용 추정 + 주의 문구`로 표현한다
- UI는 화려함보다 진행 상태와 신뢰성을 우선한다

## 8. 심사 포인트 대응

| 심사 기준 | 대응 전략 |
|---|---|
| 에이전트다운 문제 해결 | 자연어 요청 하나로 인터뷰, 브라우징, 분석, 문서 생성을 연결 |
| 어려운 문제 해결 | 감정적 고민을 재무/시장/실행 문제로 구조화해 처리 |
| 실제 동작 여부 | live 브라우저 액션과 결과 화면을 한 세션에서 시연 |
| 사용자 가치 | 퇴사 전 손익 판단과 이직 가능성을 동시에 제공 |
| 기술 완성도 | 구조화된 입력, 상태 기반 파이프라인, 결과물 생성까지 닫힌 흐름 |

### 정량 목표

- 입력 시작 후 90초 내 첫 분석 결과 노출
- 1개 채용 사이트 live 탐색 성공
- 추천 공고 Top 5 생성
- 결과 문서 1종 이상 생성

## 9. 리스크와 대응

| 리스크 | 대응 |
|---|---|
| 채용 사이트 DOM 변경 | 대상 사이트를 1곳으로 줄이고 selector fallback 준비 |
| 크롤링/자동화 불안정 | live 시연은 1곳만, 결과는 캐시/Mock 하이브리드로 보강 |
| LLM 지연 | 단계별 스트리밍 UI와 선계산 가능한 값 분리 |
| 법률/재무 오답 위험 | 참고용 추정으로 표기, 계산 근거와 주의 문구 제공 |
| 개인정보 이슈 | 데모용 가상 프로필 사용, 민감 정보 저장 최소화 |
| 시간 부족 | MVP 외 기능은 Stretch로 분리, 데모 영향 범위 축소 |

## 11. 발표용 핵심 메시지

퇴사를 고민하는 사람은 정보가 없어서가 아니라, 정보를 행동으로 바꾸기 어려워서 멈춘다.

탈출 마스터는 단순 조언 챗봇이 아니라, 사용자의 현재 상황을 이해하고 실제 웹을 탐색해 다음 행동까지 연결하는 실행형 AI 에이전트다.

---

## 12. 최종 정리

이 프로젝트의 포인트는 기능을 많이 넣는 것이 아니다.

핵심은 아래 한 문장으로 정리된다.

> "퇴사 고민을 실제 행동 가능한 계획으로 바꿔주는 브라우저 기반 AI 에이전트"

해커톤에서는 이 비전을 크게 유지하되, 실제 시연은 `퇴사 진단 -> 채용 탐색 -> 개인화 추천 -> 문서 생성`의 4단계만 안정적으로 완주하는 전략이 가장 강하다.
